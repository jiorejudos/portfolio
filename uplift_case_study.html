UpLift Web App

The purpose of this project was to design a web application for people to crowdfund donations for charities, and allow people to donate to specific causes. I focused on a site specifically for donating to Christian missions and ministries. There are not as many web apps out there focused on missions and there are specific features that are helpful for those in missions.

User Surveys

I created a survey to find out more about who would be using this app.

87%

I found that most of the people I surveyed were interested or might be interested in a Christian crowdfunding site.

75%

Many respondents would be interested in sending out e-mail updates to donors through the site.

50%

Half the respondents had used a crowdfunding site before.

Competitive Analysis

I analyzed three different apps for taking notes – GiveSendGo, YouCaring and Funded by Faith – to see what kind of features they offered.

Strengths

All three apps are free and easy to use. GiveSendGo and Funded by Faith are oriented towards specific audience, while You Caring has a wider audience.

Weaknesses

GiveSendGo requires a photo to start an account. YouCaring could use better social sharing optimization. Funded by Faith is only for churches, not for other Christian non-profits.

Opportunities

GiveSendGo and YouCaring can both send out basic emails to donors, but a site with more features for communicating with donors would be a benefit.

Threats

Offering a site that you can send out email newsletter updates.

From this research, I learned that there is a market for a well-designed Christian crowdfunding site that allows users to communicate with donors.

User Testing

As I created the prototypes and went through usability testing, I found that users liked the concept, a fundraising site aimed at missions was a unique approach. Some test participants took a little bit to figure out what the site was for, so I moved up the tagline so users see that right away. Also on the donation page, I added a progress tracker so that the user can where they are in the process. This performed well in user testing. On the homepage in the Why Uplift? section, users asked for more details, so I added information.

Project Lessons

In this project, I learned the importance of making sure the steps a user needs to take are straightforward. When an UpLift users goes through the donation process, I wanted to make sure the user knew what steps to take. I added a progress bar, to make that clearer for the donator.

In order for a crowdfunding site to be useful for a fundraiser, it really needs to make the process of getting donations easier. So I added a clear spreadsheet of who the donors were and how much they had given. This helps fundraisers keep track of donators. This page also tracks wether they have sent out a thank you note to the donor or not.
